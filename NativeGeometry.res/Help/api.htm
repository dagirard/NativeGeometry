NativeGeometry API


NativeGeometry has been developped with a consequent API documentation set, documentation generated with NativeSoft NativeDoc.
The API is categorized inside packages enabling developper to quickly find the fonction/command needed.
This help file explain quickly how NativeGeometry works and how you can use the NativeGeometry for your needs.

To open the NativeGeometry API, click here.

"Set the NativeGeometry of"


The main property of the NativeGeometry is "set/get the NativeGeometry".
If you need to update at the runtime the geometry of an object, then this command is made for you.
This command is heavily documented and the documentation is available here.

Update NativeGeometry configuration


NativeGeometry at startup set its configuration depending the platform, inside the configuration there is paddings, minimal objects size...
To read/set the configuration of NativeGeometry, there is the ng_getConfig() function and ng_setConfig command.

Run manually the NativeGeometry engine


The NativeGeometry engine is automatically executed after the "resizeStack" and the "preOpenCard" handlers, if they are passed.

If you do not want to pass those handler, or you need to automatically execute NativeGeometry, the command to launch the NativeGeometry engine on a card is ng_Run.
