Set up your application to use NativeGeometry


NativeGeometry has been designed to be the less intrusive inside your application, to make your application using NativeGeometry you have just to place the following script inside your main application stack.

----------------------------------------------
on startup
-- Opening NativeGeometry
open invisible stack "NativeGeometry"

-- On iOS, setting that the application has to use all pixel of the devices
if the platform is "iphone" then
iphoneUseDeviceResolution true, true
end if

-- Setting the scale value for this platform
ng_setConfig "scale", ng_getSystemScale()

pass startup
end startup

on savingStandalone
-- Embedding NativeGeometry
set the mainstack of stack "NativeGeometry" to "My Mainstack"
end savingStandalone

on savingMobileStandalone
-- Embedding NativeGeometry in Mobile app
set the mainstack of stack "NativeGeometry" to "My Mainstack"
save me -- Specified in android livecode note
end savingMobileStandalone

-- Optional
on preopenstack
-- Setting the font for this platform.
ng_setStackFont ng_getSystemFont(), (the name of this stack)
pass preopenstack
end preopenstack
----------------------------------------------

When the application will be "compiled", the NativeGeometry library will be embedded inside the mainstack. At the startup of the application, NativeGeometry library is opened and is ready to be used.

More information about this in the NativeGeometry API, click here to open the API.

Pass your resizeStack and preOpenCard handlers!


NativeGeometry is automatically executed after the resizeStack and the preOpenCard handlers.
If you do not want to run manually NativeGeometry do not forget to place "pass resizeStack" and "pass preOpenCard" at the end of those handlers.
